# xstack v0.3.0 milestone / xstack v0.3.0 里程碑

## Working title / 工作标题

**xstack v0.3.0 — Node-first workflow stack**  
**xstack v0.3.0 —— Node-first 工作流栈**

## Milestone summary / 里程碑摘要

xstack v0.3.0 should be the milestone where the project stops reading like a general web workflow experiment and starts reading like a clear **Node/TypeScript delivery stack for Codex**.  
xstack v0.3.0 应该成为这样一个里程碑：项目不再像一个泛化的 Web workflow 实验，而开始明确成为一个面向 Codex 的 **Node/TypeScript 交付栈**。

## What this milestone should communicate / 这个里程碑应该传达什么

By v0.3.0, xstack should feel opinionated in the right direction:  
到了 v0.3.0，xstack 应该在正确的方向上体现出足够明确的倾向：

- Node-first  
  以 Node 为中心
- TypeScript-friendly  
  对 TypeScript 友好
- web-delivery focused  
  聚焦 Web 交付
- strong on review, QA, release judgment, and stack-specific workflow support  
  在 review、QA、发版判断和技术栈专项支持上更强

## Core message / 核心信息

xstack is no longer just a compact AI workflow kit. It is becoming a practical operating layer for teams building real Node web apps with Codex.  
xstack 不再只是一个紧凑的 AI workflow 工具包，而正在变成一层实用的操作层，帮助团队用 Codex 构建真实的 Node Web 应用。

## v0.3.0 themes / v0.3.0 主题

### 1. Node-first positioning / Node-first 定位

Make the public project language more explicit about Node/TypeScript as the primary ecosystem.  
让项目的对外表达更明确地指向 Node/TypeScript 生态。

### 2. Better onboarding / 更好的上手体验

Make the first 10 minutes smoother for Node teams.  
让 Node 团队在最初 10 分钟内更容易上手。

### 3. Stronger stack-specific guidance / 更强的技术栈专项指导

Build on the packs already added:  
建立在现有技术栈 pack 的基础上：

- `nextjs-app-pack`
- `prisma-pack`
- `nestjs-backend-pack`
- `trpc-pack`
- `express-api-pack`

### 4. Better examples / 更强的示例体系

Turn examples from generic usage hints into concrete Node-first workflows.  
把 examples 从通用使用提示升级成具体的 Node-first 工作流示例。

## Proposed deliverables / 建议交付物

For v0.3.0, the minimum meaningful deliverables should be:  
对于 v0.3.0，最小但有意义的交付物应该包括：

- refined Node-first messaging in README and release materials  
  在 README 和 release 文案中强化 Node-first 表达
- expanded Node-first examples  
  扩充 Node-first 示例
- improved install/onboarding guidance  
  改善安装和上手指引
- clearer recommended workflows for common Node stacks  
  为常见 Node 技术栈提供更明确的推荐流程

## Suggested public release framing / 建议的对外发布表述

### Short version / 短版

xstack v0.3.0 is the release where xstack becomes clearly Node-first.  
xstack v0.3.0 是 xstack 明确转向 Node-first 的版本。

### Longer version / 长版

xstack v0.3.0 turns xstack from a strong web workflow baseline into a clearer Node/TypeScript delivery stack for Codex, with more concrete stack support, better examples, and stronger onboarding.  
xstack v0.3.0 将 xstack 从一个强力的 Web workflow 基线，推进成一个更清晰的面向 Codex 的 Node/TypeScript 交付栈，具备更具体的技术栈支持、更好的示例体系和更强的上手体验。

## Success criteria / 成功标准

v0.3.0 should make it easier for a Node team to:  
v0.3.0 应该让一个 Node 团队更容易做到：

- understand what xstack is for  
  理解 xstack 是干什么的
- install and try it quickly  
  快速安装并试用
- choose the right skills and packs  
  选对该用的 skills 和 packs
- apply it to a real stack such as Next.js, Prisma, NestJS, tRPC, or Express  
  把它用于真实技术栈，如 Next.js、Prisma、NestJS、tRPC 或 Express

## Not the goal / 不属于这个里程碑的目标

v0.3.0 should not try to support every ecosystem.  
v0.3.0 不应该试图支持所有生态。

It should deepen the Node-first path rather than widen the matrix.  
它应该把 Node-first 这条路线做深，而不是把支持矩阵继续做宽。
